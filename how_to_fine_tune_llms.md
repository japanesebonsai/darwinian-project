# Fine-Tuning an LLM with Darwinian Council Data

This guide explains how to fine-tune a model (like Llama-3 or Mistral) to specialize in project ideation and critical debate, specifically using the output generated by the Darwinian Council repository.

## 1. Why Fine-Tune on This Repository?

The Darwinian Council generates high-quality, multi-agent debates with strict formatting (XML tags for `<thought>`, `<pitch>`, `<scores>`, and `<critique>`). By fine-tuning an open-weights model on this data, you can create a specialized "Ideation Model" that natively outputs this format and has internalized the critical, multi-perspective debate mechanics.

## 2. Extracting and Formatting the Data

Currently, the council stores its state using Zustand (`rounds` array). To fine-tune, you need to collect these debates and format them into a standard training structure (like ChatML).

**Example Extraction Script:**
You can add a feature to `app/council/page.tsx` to export `store.rounds` as a JSONL file. 

The format should map the user's initial prompt (Problem, Audience, Constraints) to the system prompt, and the agent's output as the assistant response.

```json
{"messages": [
  {"role": "system", "content": "You are a participant in a project ideation council... Your persona: [Dynamic Persona]"},
  {"role": "user", "content": "PROBLEM: Build a transparency ledger. AUDIENCE: Web3 skeptics. CONSTRAINTS: No blockchain."},
  {"role": "assistant", "content": "<thought>We need trust without blockchain...</thought>\n<pitch>A cryptographically signed, append-only SQLite database with public key verifications.</pitch>\n<scores>...</scores>\n<critique>...</critique>"}
]}
```

## 3. Training with Unsloth

Once you have at least 500-1000 high-quality agent responses exported in JSONL format, you can use **Unsloth** (via Google Colab or a local GPU).

1. **Install Unsloth:** Follow instructions on the Unsloth GitHub repository.
2. **Load a Base Model:** Start with a good reasoning model (e.g., `unsloth/llama-3-8b-Instruct-bnb-4bit`).
3. **Configure LoRA:** Apply adapters to the attention blocks.
4. **Train:** Feed your exported ChatML JSONL file into the `SFTTrainer`. Use a low learning rate (`2e-5`) for 1-3 epochs.

> [!TIP]
> **Focus on Formatting:** LLMs learn format very quickly. The main benefit of fine-tuning here is teaching the model to always return the correct XML structure (`<thought>`, `<pitch>`, etc.) without relying on massive system prompts or prompt engineering.

## 4. Deploying the Fine-Tuned Model

Once fine-tuned, you can export the LoRA adapters or merge them into the base model (creating GGUF files for local execution via Ollama/LM Studio). 

You can then update `lib/models.ts` in this project to point to your local endpoint (e.g., `http://localhost:11434/v1`) and use your newly fine-tuned Darwinian Ideation Model instead of OpenRouter models!
